Frequently Asked Questions

1. What is the difference between var, let, and const?
var, let, and const are JavaScript keywords for variable declaration, differing mainly in scope, reassignment, and hoisting: var is function/global-scoped, re-declareable/reassignable; let is block-scoped, reassignable but not re-declareable; const is block-scoped, reassignable only if the property of an object (not the variable itself) and not re-declareable, requiring initialization. Modern JavaScript favors let for mutable variables and const for constants, avoiding var due to its less predictable scope.
2. What is the difference between map(), forEach(), and filter()?
. map() is used when you want to transform each element of an array. It runs a function on every item and returns a new array with the modified values, without changing the original array. . forEach() is used to perform an action on each element of an array, such as logging values or updating the UI. It does not return anything, so it is mainly used for side effects rather than data transformation. . filter() is used when you want to select specific elements from an array based on a condition. It checks each element and returns a new array containing only the elements that pass the condition.
4. How do JavaScript Promises work??
JavaScript Promises are used to handle asynchronous operations. A Promise represents a value that may be available now, in the future, or never. A Promise has three states: pending (initial state), fulfilled (operation completed successfully), and rejected (operation failed). When a Promise is fulfilled, it returns a result; when rejected, it returns an error. You handle these results using .then() for success and .catch() for errors, which helps write cleaner and more readable asynchronous code compared to traditional callbacks.
5. How does closure work in JavaScript
A closure in JavaScript happens when a function remembers and can access variables from its outer (lexical) scope, even after the outer function has finished executing. This works because functions in JavaScript keep a reference to the scope in which they were created. As a result, inner functions can continue to use outer variables, which is commonly used for data encapsulation, maintaining state, and creating private variables.
3. What are arrow functionsy and now are they different from regular functions?
Arrow functions are a shorter syntax for writing functions in JavaScript. They are commonly used for cleaner and more readable code. They are different from regular functions in a few important ways. Arrow functions do not have their own this keyword; instead, they inherit this from the surrounding scope. This makes them especially useful in callbacks. They also do not have their own arguments object and cannot be used as constructors with the new keyword. Regular functions, on the other hand, have their own this, support arguments, and can be used as constructors.
